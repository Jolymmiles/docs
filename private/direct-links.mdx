title: Browser login (direct links)
description: Enable direct browser access via t.me links
---

Use this setup to let you open the Mini App directly in a browser via `t.me` links (without the in-app Telegram browser).

## Configure domain in BotFather

1. Open [@BotFather](https://t.me/BotFather)
2. Send the command `/setdomain`
3. Select your bot from the list
4. Enter your admin domain (for example, `bot.example.com`)

<Note>
The domain must match the `BOT_ADMIN_URL` value (without the `https://` prefix).
</Note>

After this configuration, users can open the Mini App via direct browser links.

## Supported startapp parameters

Use Telegram deep links to open specific sections inside the Mini App:

```
https://t.me/<bot_username>?startapp=<value>
```

| `startapp` value | Opens | Example |
|---|---|---|
| *(empty)* | Default app entry (redirects based on role) | `https://t.me/<bot_username>?startapp` |
| `user_<telegram_id>` | Admin user details | `https://t.me/<bot_username>?startapp=user_123456789` |
| `support_ticket_<ticket_id>` | Admin support ticket | `https://t.me/<bot_username>?startapp=support_ticket_42` |
| `buy_<plan_id>_<months>` | User payment flow for a plan period | `https://t.me/<bot_username>?startapp=buy_12_3` |
| `sub_<subscription_id>` | User dashboard for a specific subscription | `https://t.me/<bot_username>?startapp=sub_98765` |

## Direct browser links (https://bot.example.com)

The Mini App uses hash-based routes. Use the `/#/` prefix in direct links:

```
https://bot.example.com/#/<path>
```
